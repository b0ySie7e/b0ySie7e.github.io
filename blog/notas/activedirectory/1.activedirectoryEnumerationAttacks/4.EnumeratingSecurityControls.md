## Controles de Seguridad

### Windows Defender

```c
Get-MpComputerStatus
```

Revisa `RealTimeProtectionEnabled`, `AMServiceEnabled`, `IsTamperProtected`. Si están en `True`, el AV está activo y hay que buscar bypass antes de ejecutar herramientas.

### AppLocker

```c
Get-AppLockerPolicy -Effective | select -ExpandProperty RuleCollections
```

Si bloquea PowerShell, intentar rutas alternativas como `%SystemRoot%\SysWOW64\WindowsPowerShell\v1.0\powershell.exe`.

### PowerShell Constrained Language Mode

```c
$ExecutionContext.SessionState.LanguageMode
```

`ConstrainedLanguage` = muchas funciones bloqueadas. Buscar alternativas con binarios nativos.

### LAPS

```c
# Ver OUs con LAPS y grupos con delegación
Find-LAPSDelegatedGroups

# Ver quién tiene Extended Rights (puede leer passwords LAPS)
Find-AdmPwdExtendedRights

# Leer passwords LAPS si tienes acceso
Get-LAPSComputers
```

## Enumeración con Credenciales — desde Linux

### CrackMapExec

```c
# Usuarios del dominio
sudo crackmapexec smb 172.16.5.5 -u forend -p Klmcargo2 --users

# Grupos del dominio
sudo crackmapexec smb 172.16.5.5 -u forend -p Klmcargo2 --groups

# Usuarios con sesión activa en un host
sudo crackmapexec smb 172.16.5.130 -u forend -p Klmcargo2 --loggedon-users

# Shares disponibles
sudo crackmapexec smb 172.16.5.5 -u forend -p Klmcargo2 --shares

# Spider de shares — indexa todos los archivos accesibles
sudo crackmapexec smb 172.16.5.5 -u forend -p Klmcargo2 -M spider_plus --share 'Department Shares'

# Ver el resultado del spider
head -n 10 /tmp/cme_spider_plus/172.16.5.5.json

# Con proxychains y netexec (alternativa moderna a CME)
proxychains netexec ldap 172.16.5.5 -u forend -p Klmcargo2 --groups 2>/dev/null
proxychains netexec smb 172.16.5.5 -u forend -p Klmcargo2 -M spider_plus --share 'Department Shares'
```

### smbmap

```c
# Ver shares y permisos
smbmap -u forend -p Klmcargo2 -d INLANEFREIGHT.LOCAL -H 172.16.5.5

# Listar directorios recursivamente (solo dirs)
smbmap -u forend -p Klmcargo2 -d INLANEFREIGHT.LOCAL -H 172.16.5.5 -R 'Department Shares' --dir-only

# Versión -r (minúscula) en versiones modernas
smbmap -u forend -p Klmcargo2 -d INLANEFREIGHT.LOCAL -H 172.16.5.5 -r 'Department Shares' --dir-only
```

### rpcclient con credenciales

```c
rpcclient -U "" -N 172.16.5.5
rpcclient $> enumdomusers
rpcclient $> queryuser 0x457
rpcclient $> enumdomgroups
rpcclient $> querygroup 0x201
```

### Impacket — Ejecución remota

```c
# Shell interactiva via SMB (similar a PsExec)
psexec.py inlanefreight.local/wley:'transporter@4'@172.16.5.125

# Ejecución remota via WMI (más silencioso que psexec)
wmiexec.py inlanefreight.local/wley:'transporter@4'@172.16.5.5
```

### windapsearch

```c
# Domain Admins
python3 windapsearch.py --dc-ip 172.16.5.5 -u forend@inlanefreight.local -p Klmcargo2 --da

# Usuarios privilegiados (todos los grupos admin)
python3 windapsearch.py --dc-ip 172.16.5.5 -u forend@inlanefreight.local -p Klmcargo2 -PU

# Todos los usuarios
python3 windapsearch.py --dc-ip 172.16.5.5 -u forend@inlanefreight.local -p Klmcargo2 -U
```

### BloodHound.py

```c
# Recolección completa con nameserver explícito
bloodhound-python -u 'forend' -p 'Klmcargo2' -ns 172.16.5.5 -d inlanefreight.local -c all

# Ver opciones disponibles
bloodhound-python -h

# Alternativa con -dc
bloodhound-python -d INLANEFREIGHT.LOCAL -dc ACADEMY-EA-DC01 -c All -u forend -p Klmcargo2

# Comprimir para subir a BloodHound
zip -r ilfreight_bh.zip *.json

# Iniciar base de datos Neo4j para BloodHound
sudo neo4j start
```

## Enumeración con Credenciales — desde Windows

### Módulo ActiveDirectory (cmdlets nativos)

```c
# Cargar módulo
Import-Module ActiveDirectory
Get-Module

# Info del dominio
Get-ADDomain

# Usuarios con SPN (Kerberoasteables)
Get-ADUser -Filter {ServicePrincipalName -ne "$null"} -Properties ServicePrincipalName

# Trusts del dominio
Get-ADTrust -Filter *

# Grupos del dominio
Get-ADGroup -Filter * | select name

# Info detallada de un grupo
Get-ADGroup -Identity "Backup Operators"

# Miembros de un grupo
Get-ADGroupMember -Identity "Backup Operators"
```

### PowerView — Enumeración completa

```c
Import-Module .\PowerView.ps1

# Info del dominio y DCs
Get-Domain
Get-DomainController | select Name, IPAddress
Get-DomainPolicy

# Usuario específico con propiedades clave
Get-DomainUser -Identity mmorgan -Domain inlanefreight.local | Select-Object -Property name,samaccountname,description,memberof,whencreated,pwdlastset,lastlogontimestamp,accountexpires,admincount,userprincipalname,serviceprincipalname,useraccountcontrol

# Todos los usuarios con descripción (buscar contraseñas)
Get-DomainUser * | Select-Object samaccountname,description | Where-Object {$_.Description -ne $null}

# Usuarios con SPN
Get-DomainUser -SPN -Properties samaccountname,ServicePrincipalName

# Usuarios sin pre-autenticación Kerberos (ASREPRoasteable)
Get-DomainUser -PreauthNotRequired | select samaccountname

# Membresía de grupo recursiva (incluye grupos anidados)
Get-DomainGroupMember -Identity "Domain Admins" -Recurse

# Mapeo de trusts
Get-DomainTrustMapping

# Verificar acceso de admin local en un host
Test-AdminAccess -ComputerName ACADEMY-EA-MS01

# Buscar shares accesibles en el dominio
Find-DomainShare
Find-InterestingDomainShareFile -Include *.ps1,*.bat,*.txt

# Buscar máquinas donde el usuario actual tiene admin local
Find-LocalAdminAccess

# Buscar dónde están conectados usuarios específicos
Find-DomainUserLocation -UserGroupIdentity "Help Desk Level 1"

# GPOs del dominio
Get-DomainGPO | select displayname
```

### SharpView (versión .NET de PowerView)

```c
# Ver ayuda de un comando
.\SharpView.exe Get-DomainUser -Help

# Obtener info de un usuario específico
.\SharpView.exe Get-DomainUser -Identity forend
```

### SharpHound — Recolectar datos para BloodHound

```c
# Ver opciones
.\SharpHound.exe --help

# Recolección completa
.\SharpHound.exe -c All --zipfilename ILFREIGHT

# Con módulo PowerShell
Import-Module .\Sharphound.ps1
Invoke-BloodHound -CollectionMethod All -OutputDirectory C:\Users\forend.INLANEFREIGHT\Desktop\ -OutputPrefix "ilfreight"
```

### Snaffler — Buscar archivos con credenciales en shares

```c
.\Snaffler.exe -s -d inlanefreight.local -o snaffler.log -v data
.\Snaffler.exe -d INLANEFREIGHT.LOCAL -s -v data
```

`-s` muestra output en consola, `-d` especifica el dominio, `-o` guarda a archivo, `-v data` muestra archivos de interés.

## BloodHound — Queries Cypher útiles

```c
# Usuarios con acceso WinRM (CanPSRemote)
MATCH p1=shortestPath((u1:User)-[r1:MemberOf*1..]->(g1:Group)) MATCH p2=(u1)-[:CanPSRemote*1..]->(c:Computer) RETURN p2

# SQL Admin rights
MATCH p1=shortestPath((u1:User)-[r1:MemberOf*1..]->(g1:Group)) MATCH p2=(u1)-[:SQLAdmin*1..]->(c:Computer) RETURN p2
```

Queries pre-construidas relevantes en el panel Analysis:

- `Find Workstations where Domain Users can RDP`
- `Find Servers where Domain Users can RDP`
- `Find Computers with Unsupported Operating Systems`
- `Users with Foreign Domain Group Membership`
- `Map Domain Trusts`
